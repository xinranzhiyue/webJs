请首先打开seasons's_song.html导航页面，这个是首页。

请使用chrome或火狐高版本打开本页面，以得到最佳体验。


四季之歌这个构思来源于对家乡的回忆，本来的想法是通过四季的变换，描绘出一个简单美丽的小县城，但考虑到认知的接受程度，将重点从地域改为了季节。

之所以选择5个页面来构成设计，一方面是由于需要控制的元素较多，写在同一个页面的话，单单命名就会很头疼，更不要说还需要考量定位，层级。。。；另一方面，既然名字叫做四季之歌，就要每一个季节配一个适合的bgm，当然，也可以写一个能够根据页面切换，自动切换bgm的音乐插件，不过现在的我实在无能为力，只能选择“优雅降级”了。

seasons's_song.html
这个页面的切换动画类似推拉门，滚轮上下滑动，就像打开，关闭大门一样，描绘了春夏秋冬的循环；
点击标题跳转跳转到相对应的页面；

背景是一套四季系列的图片，但没有找到无字的原图，只能用ps抹掉原图的字迹，不过由于技术原因，只能说“毫无ps痕迹”了；

点击小猫logo可以跳转到我的github页面，里面有一些个人做的其他页面，不过大多是静态的；

除autumn.html外，其他页面利用百分比和vw，vh基本上做到了自适应屏幕；

js代码中原生js与jquery代码都有使用，主要是为了练习及适应不同情况；

本页面对应js代码 seasonsLoop.js。


spring.html

利用css3中的3d属性，构建的一个3d魔方，确切来说是两个3d魔方――一个里面嵌套另一个；

本页面对应js代码 3DPhotoWall.js。


summer.html

由于canvas的宽高特性，很难实现自适应屏幕大小的同时，其构建的图片不会拉伸，因此本页面没有实现自适应，以后对canvas有更深理解的时候，再进行修改吧。建议在1366 x 768分辨率下浏览；

使用canvas的蒙版制作了一个伪动画背景，添加了拖拽的回放，如果没有吧萤火虫放入瓶子里，萤火虫会按照原路线返回；

本页面对应js代码 fireFly.js。


autumn.html

利用了css3的动画关键帧，即@keyframes属性制作出飘零的落叶。小游戏部分则是原生js控制；

两者区别在于一个是旋转下路，一个是直线坠落；

本页面对应js代码 fallingLeaves.js。


winter.html

雪人是完全由css代码构成的，雪花则是由js代码生成的；

移动鼠标到雪人，雪人某些部分会变色，搭配出最喜爱的雪人吧；

解决不了的bug：改变浏览器窗口大小时，雪花下落速度变得很快且不可控；

本页面对应js代码 sonwMan.js。


兼容性

兼容火狐及chrome高版本，理论上支持所有使用chrome核心的浏览器；

不兼容safari及ie（所有版本）。
